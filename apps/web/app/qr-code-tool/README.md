这个工具用于在当前浏览器页面中完成二维码的生成与解析，支持 `文本`、`网址`、`Wi-Fi`、`电话`、`短信`、`邮箱` 等常见内容类型，并提供 `Logo`、`透明背景`、容错等级、图片下载和二维码识别能力，适合活动物料制作、设备配网、办公分享和扫码排障。

## 功能说明

### 生成二维码

1. 支持 `文本`、`网址`、`Wi-Fi`、`电话`、`短信`、`邮箱` 六类常见内容
2. 支持设置二维码尺寸、边距、前景色、背景色和透明背景
3. 支持切换主体码点样式、定位角样式和容错等级
4. 支持上传中间 `Logo / 图片`，并即时刷新预览
5. 支持导出 `PNG`、`SVG`、`JPEG`、`WebP`
6. 支持复制二维码图片和复制原始编码内容

### 解析二维码

1. 支持上传图片解析二维码
2. 支持拖拽二维码图片到上传区域
3. 支持复制截图后直接粘贴识别
4. 自动识别网址、Wi-Fi、电话、短信、邮箱和普通文本
5. 展示结构化字段和快捷操作入口

## 使用说明

### 生成模式

1. 在左侧选择内容类型
2. 按类型填写对应字段
3. 按需设置颜色、透明背景、Logo 和容错等级
4. 在右侧实时查看二维码预览和可读性提示
5. 点击复制图片、下载图片或复制原始内容

### 解析模式

1. 切换到“解析二维码”
2. 上传、拖拽或粘贴二维码图片
3. 在右侧查看识别出的结构化结果
4. 按需复制原始值，或使用快捷动作打开链接、邮件或拨号

## 安全说明

1. 二维码生成、样式渲染和图片解析都仅发生在当前浏览器页面内
2. 不会主动将二维码内容、Logo 图片或截图上传到服务器
3. 不会默认保存历史记录或本地草稿
4. 页面刷新或关闭后，当前结果会随会话一起清空
5. 如果你主动复制图片、下载二维码或打开外部链接，后续数据将由浏览器和操作系统接管

## 限制说明

- 当前首版主要覆盖静态二维码，不包含动态二维码跳转统计和有效期管理
- 图片解析当前以单张二维码为主，多码图片默认只识别其中一个成功结果
- 二维码过度美化、颜色对比过低、Logo 过大或边距不足时，实际扫码成功率会下降
- 当前解析模式优先处理图片文件和剪贴板截图，不包含摄像头实时扫码
- Wi-Fi、短信和邮箱二维码的实际唤起效果取决于用户设备和系统对协议的支持情况

## 常见问题

### 为什么带 Logo 的二维码还能扫

二维码自带容错能力。只要 `Logo` 占比、容错等级和边距设置合理，扫码器仍然可以恢复原始数据。

### 透明背景二维码是否可靠

可以使用，但更依赖承载它的底色和对比度。建议将透明二维码放在纯色浅底上，避免复杂图片背景。

### Wi-Fi 二维码支持哪些字段

当前支持 `SSID`、`加密方式`、`密码` 和 `隐藏网络`。生成内容遵循常见的 `WIFI:T:...;S:...;P:...;;` 格式。

### 为什么有些二维码图片识别失败

常见原因包括：截图过小、图片压缩严重、二维码被裁切、颜色反差过低、Logo 过大或图片中包含强烈透视变形。

### 为什么网址二维码要提示 HTTP 风险

如果二维码指向 `HTTP` 非加密链接，公开传播时更容易遭遇中间人劫持或跳转风险，因此工具会给出提醒。
