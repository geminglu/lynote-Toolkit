这个工具用于在当前浏览器页面中即时转换和微调常见 CSS 颜色格式，适合前端开发、设计协作、主题联调和组件样式排查。

## 功能说明

### 颜色格式

1. `HEXA`
2. `RGBA`
3. `HSLA`
4. `HWB`
5. `lab()`
6. `lch()`
7. `oklch()`

### 交互能力

1. 任意格式修改后，其他格式会实时同步
2. 每种格式都支持完整字符串输入、通道输入和滑块调节
3. 每种格式都可以打开颜色选择器并调整透明度
4. 每种格式都可以单独复制当前规范化结果

### 单一颜色状态联动

页面内部只维护一份统一颜色状态。你无论修改 `RGBA`、`HSLA`、`HWB`、`lab()`、`lch()` 还是 `oklch()`，其他模块都会立即同步更新。

### 颜色选择器

每个格式模块都可以展开同一个颜色选择能力：

- 使用色板选择基础色相
- 使用透明度输入框和滑块控制 `alpha`
- 修改后同步刷新全部颜色模块

### 透明度预览

当前页面只展示：

- 当前颜色本身
- 透明棋盘格背景下的预览效果

不会额外展示白底、黑底等“不透明背景预览”。

### 规范化输出

每个模块都会输出适合复制的规范化字符串，例如：

- `#7C3AEDD1`
- `rgba(124, 58, 237, 0.82)`
- `hsl(270 83.9% 58.8% / 0.82)`
- `oklch(68% 0.22 285 / 0.82)`

## 使用说明

1. 打开页面后先查看左侧当前颜色预览。
2. 在右侧任意一个格式模块中输入颜色字符串，或直接拖动通道滑块。
3. 如果需要更直观地选色，点击该模块内的“打开”按钮展开颜色选择器。
4. 修改任一模块后，其余格式会自动同步。
5. 点击“复制”即可复制当前模块的规范化结果。
6. 点击“恢复默认颜色”可回到初始状态。

## 安全说明

本工具仅在当前浏览器页面内存中处理颜色值和转换结果。

1. 不会把颜色值上传到服务器。
2. 不会自动写入本地存储。
3. 不会保存历史颜色记录。
4. 如果你主动复制结果，颜色字符串将由浏览器和操作系统接管。

## 限制说明

- 当前首版不支持 `CMYK`、`HSV / HSB`、`color(display-p3 ...)` 等格式。
- `lab()`、`lch()`、`oklch()` 与 `RGB` 系颜色之间转换时可能存在小数舍入。
- 颜色选择器的基础色板只负责选择颜色本体，透明度仍通过独立通道控制。

## 常见问题

### 为什么不同格式的数字看起来差很多

因为不同颜色模型表达颜色的方式不同：

- `RGBA` 更接近屏幕通道
- `HSLA` 更适合围绕色相和亮度调色
- `Lab / LCH / OKLCH` 更接近感知均匀空间

它们描述的是同一颜色，但数值体系并不相同。

### 为什么 `lab()` 或 `oklch()` 改完后 `RGBA` 会出现微小误差

这是不同颜色空间互转时的正常现象。页面会输出规范化后的结果，便于直接复制到代码中使用。

### 这个工具适合什么场景

适合：

- 调整组件主题色
- 联调设计稿里的 `oklch()`
- 把设计系统颜色换成浏览器可直接使用的 `rgba()` / `hsl()`
- 排查透明色在不同表示方式下的结果
