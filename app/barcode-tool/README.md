这个工具用于在当前浏览器页面中完成条形码的生成与解析，支持 `CODE128`、`EAN-13`、`EAN-8`、`UPC-A`、`UPC-E`、`Code39`、`Code93`、`ITF-14`、`Interleaved 2 of 5`、`Codabar`、`DataMatrix`、`PDF417`、`Aztec` 等常见码制，适合物流标签、商品编码、票务条码和工业追溯场景。

## 功能说明

### 生成条形码

1. 支持 13 种常见一维码与二维数据码（不含 QR，QR 请使用 [二维码工具](/qr-code-tool)）
2. EAN-13、EAN-8、UPC-A、UPC-E、ITF-14 等数字码自动计算校验位
3. 支持调整缩放倍数、一维码高度、边距、前景/背景色、透明背景
4. 支持显示人眼可读文字，并可选放在条形码的上方或下方
5. 支持导出 `PNG` 与 `SVG`
6. 支持复制条形码图片和复制原始内容

### 解析条形码

1. 支持上传图片解析条形码
2. 支持拖拽条形码图片到上传区域
3. 支持复制截图后直接粘贴识别
4. 自动识别码制类型并展示原始字符串
5. 支持 CODE128、EAN、UPC、Code39、ITF、DataMatrix、PDF417、Aztec 等多种码制的识别

## 使用说明

### 生成模式

1. 在左侧选择目标码制
2. 按格式要求填写内容（数字码会标注位数与校验位规则）
3. 按需调整缩放、高度、边距、颜色和可读字位置
4. 在右侧实时预览生成结果与提示信息
5. 点击复制图片、下载图片或复制原始内容

### 解析模式

1. 在左侧切换为“解析条形码”
2. 上传、拖拽或粘贴条形码图片
3. 在右侧查看识别出的码制类型和原始字符串
4. 点击复制原始内容，继续接入其他工具进行验证或对账

## 安全说明

1. 条形码内容、渲染图片和识别图片都仅在当前浏览器页面执行
2. 不会主动上传原始内容、条形码图片或截图到任何服务器
3. 下载图片或复制内容后，数据将由浏览器和操作系统接管

## 限制说明

- 不包含 QR 码，QR 场景请使用 [二维码工具](/qr-code-tool)
- 摄像头实时扫码暂未提供，仅支持图片上传/拖拽/粘贴识别
- 大于 12MB 的图片会被拒绝以避免浏览器卡顿
- 部分罕见码制（如全部 GS1 复合码、邮政码）暂未在 UI 中开放
- 旋转方向仅影响生成输出，扫码识别始终按原图方向进行

## 常见问题

### 为什么 EAN-13 输入 12 位也能生成？

EAN-13 的第 13 位是校验位，工具会按 GS1 规则自动计算。如果直接输入 13 位，系统会校验最后一位与计算结果是否一致。

### 为什么生成的 PNG 太小？

可以调大“缩放倍数”，或者保持当前缩放并改为下载 `SVG` 以无损放大。

### 解析失败怎么排查？

优先上传更清晰的原图，保证条形码两端的静区完整；其次确认图片没有过强的高对比度滤镜，避免线宽丢失。

### 工具支持中文吗？

DataMatrix、PDF417、Aztec 等二维数据码支持任意 UTF-8 文本；一维码（CODE128 等）仅支持各自规范内的字符集。
