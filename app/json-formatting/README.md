这个工具用于校验、格式化、压缩、转义、递归排序 key 和查看 JSON 内容，适合处理原始 JSON、转义后的 JSON 字符串，以及普通文本转 JSON 字符串的场景。

1. 普通 JSON

```json
{ "name": "tool", "enabled": true }
```

2. 已转义的 JSON 字符串

```text
"{\"name\":\"tool\",\"enabled\":true}"
```

3. 外层带引号的 JSON 文本

```text
'{"name":"zs","targetHost":"https://xxx.sss.com.cn"}'
```

4. 普通字符串

```text
hello world
```

输入普通字符串时，右侧会立即转换为合法的 JSON 字符串格式，例如：

```json
"hello world"
```

## 使用说明

1. 在左侧输入原始内容，支持 JSON、JSON 字符串和普通文本。
2. 当内容可以被识别和解析时，右侧会自动展示格式化结果。
3. 如果左侧输入非法 JSON，页面顶部会显示错误提示，右侧不会更新。
4. 你可以使用工具栏对左右两侧分别执行格式化、key 排序、压缩、转义、复制和下载操作。
5. 上传文件时，内容会默认导入到左侧主输入区。
6. 新建草稿后不会立即进入历史记录，只有开始编辑后才会自动保存。
7. key排序，默认未排序，第一次点击按 key 升序排序，再点一次切换为 key 降序排序。

## 适用场景

### 接口联调

当你在排查请求体、响应体、Webhook 回调或第三方平台返回值时，可以先把原始 JSON 粘贴进来，快速确认：

- JSON 是否合法
- 是否存在转义层级问题
- 字段顺序是否影响人工比对
- 某些平台返回的字符串内容是否其实仍然是 JSON

### 配置和日志排查

很多环境变量、配置中心或日志系统中的 JSON 会被压成一行，甚至被二次转义。这个工具适合快速把它们还原成更容易阅读和复制的结构。

### 文本转 JSON 字符串

如果你需要把一段普通文本安全地嵌入 JSON 字段，例如请求体模板、脚本配置或测试数据，这个工具可以直接把普通文本转换成合法的 JSON 字符串。

## 常见问题

### 为什么我输入的是 JSON，页面却提示解析失败

常见原因包括：

- 多了尾逗号
- 字符串没有用双引号包裹
- 外层额外包了一层单引号
- 文本中混入了非法换行或不可见字符

### JSON key 排序有什么用

`key` 排序更适合做：

- 配置差异对比
- Git 变更审查
- 接口返回值人工核对
- 需要稳定字段顺序的测试样例整理

### 为什么普通字符串会被转换成带引号的结果

因为右侧输出的是合法 JSON 格式。普通文本本身不是 JSON 值，但可以被表示为 JSON 字符串，所以会自动补上引号并处理转义字符。
