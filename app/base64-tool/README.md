这个工具用于在当前浏览器页面中完成 `Base64`、`Base64URL`、`JWT` 片段、`Data URL`、文本与文件内容之间的转换和预览，适合接口联调、资源内联、JWT 排查、批量逐行处理和二进制字节检查。

## 功能说明

### 输入模式

1. `文本`
2. `Base64`
3. `Base64URL`
4. `JWT 片段`
5. `Data URL`
6. `文件`

### 输出模式

1. `Base64`
2. `Base64URL`
3. `文本`
4. `Hex`
5. `Data URL`

### 核心能力

1. 输入变化后右侧结果会自动刷新，无需手动提交
2. 支持忽略空白字符和自动补齐 `=` padding
3. 支持切换 `UTF-8`、`UTF-16LE`、`Windows-1252`、`GBK`、`Shift_JIS` 文本预览字符集
4. 支持逐行处理模式，适合一行一个值的批量转换
5. 支持文件拖拽或上传，并展示文件大小、类型和最后修改时间
6. 支持解析 `Data URL`，拆出 `mime type`、负载长度和是否为 `base64`
7. 支持查看标准 `Base64`、`Base64URL`、文本预览、Hex 预览和字节表格
8. 支持分别复制和下载主输出及附加输出

## 使用说明

1. 在左侧选择输入模式
2. 选择主输出模式
3. 如需查看文本内容，可按需切换文本预览字符集
4. 如果当前输入来自 JWT、URL 参数或被换行分隔的批量数据，可按需开启对应开关
5. 文本、Base64、Base64URL、JWT 片段和 Data URL 可直接粘贴到输入框
6. 文件模式下可点击选择文件，或直接拖拽文件到上传区域
7. 在右侧查看主输出、附加输出、Data URL 信息和字节预览
8. 你可以对任一输出执行复制或下载

## 适用场景

### JWT 片段排查

当你只想快速看 `JWT` 的 header、payload 或 signature 段内容时，可以直接选择 `JWT 片段` 模式，工具会按 `Base64URL` 规则处理输入。

### 前端资源内联

当你需要把图片、小字体或其他文件转成 `Data URL` 嵌入页面时，文件模式和 `Data URL` 输出会更方便。

### 批量逐行转换

当你拿到的是“一行一个值”的批量数据，例如日志导出、CSV 单列内容或一组签名片段，逐行处理模式可以帮助你一次性完成编码或解码。

### 二进制排查

如果解码后的内容不是正常文字，工具会保留 `Hex` 结果、ASCII 预览和字节表格，帮助你确认它到底是文本、图片、压缩包还是其他二进制内容。

## 安全说明

1. 所有编码、解码、文件读取和字节预览都仅发生在当前浏览器页面内
2. 不会主动将输入内容上传到服务器
3. 不会默认写入本地存储或保存历史记录
4. 刷新或关闭页面后，当前内容会被清空
5. 如果你主动执行复制或下载，后续数据将由浏览器和操作系统接管

## 限制说明

- 文本输入当前统一按 `UTF-8` 编码为字节；字符集切换主要影响“字节转文本”的预览结果
- 逐行处理当前仅支持 `文本`、`Base64`、`Base64URL` 与 `JWT 片段` 模式
- 非 `base64` 的 `Data URL` 当前按文本负载处理，不会保留原始百分号编码形式
- 首版仅支持单文件，不支持目录或多文件批量处理
- 过大的文件会受到浏览器内存和页面渲染能力限制

## 常见问题

### `Base64` 和 `Base64URL` 有什么区别

`Base64URL` 是适合 URL、文件名和 JWT 的安全变体，通常会把 `+` 改成 `-`，把 `/` 改成 `_`，并且经常去掉尾部的 `=`。

### 为什么有些内容没有 `=`

很多 `JWT` 或 URL 参数里的 `Base64URL` 都会省略 padding。你可以开启“自动补齐补位”后再试一次。

### 为什么解码后不是正常文字

这通常说明当前内容本来就不是文本，或者它需要按其他字符集解释。你可以切换文本预览字符集，或者直接查看 `Hex` 视图。

### 为什么我在文件模式下看不到逐行处理

逐行处理适合一行一个字符串值，而文件模式面对的是整段原始字节，所以当前版本会自动禁用逐行处理。

### 为什么 `Data URL` 输出的 `mime type` 和原始输入不同

当输入不是文件或原始 `Data URL` 时，工具会优先使用当前配置的 `Data URL MIME Type` 来生成输出，方便你按目标场景调整。

## 代码示例

### Node.js 中把文本转成 `Base64`

```ts
const text = "hello world";
const base64 = Buffer.from(text, "utf8").toString("base64");

console.log(base64);
```

### Node.js 中把 `Base64URL` 还原为文本

```ts
const input = "SGVsbG8tV29ybGQ";
const normalized = input.replace(/-/g, "+").replace(/_/g, "/");
const padded = normalized.padEnd(Math.ceil(normalized.length / 4) * 4, "=");

console.log(Buffer.from(padded, "base64").toString("utf8"));
```
