这个工具用于在当前浏览器页面中完成 `JSON`、`YAML`、`XML` 之间的数据转换，并基于左侧输入即时生成 `TypeScript`、`Zod`、`Java`、`Go`、`C` 模型代码，适合接口联调、配置迁移、示例数据整理和结构草稿生成。

## 功能说明

### 左侧输入类型

1. `JSON`
2. `YAML`
3. `XML`

### 右侧输出类型

1. `JSON`
2. `YAML`
3. `XML`
4. `TypeScript`
5. `Zod`
6. `Java`
7. `Go`
8. `C`

### 核心能力

1. 左侧主输入区编辑后，右侧结果会自动刷新
2. 左侧切换输入类型时，会清空当前内容，避免混淆输入语义
3. 右侧可切换目标类型，并即时查看转换结果或代码草稿
4. 左右两侧都支持复制、下载和格式化
5. 左侧支持上传 `.json`、`.yaml`、`.yml`、`.xml` 文件
6. 代码类型支持自定义根类型名称
7. 支持配置 XML 属性前缀、文本节点名和单节点强制数组策略
8. 支持切换 TypeScript `interface/type`、Zod `infer` 输出和 Go `json` tag

## 使用说明

1. 在左侧选择输入类型：`JSON`、`YAML` 或 `XML`
2. 粘贴内容，或上传对应格式文件
3. 在右侧选择目标类型
4. 若输入或输出涉及 XML，可按需调整 XML 规则
5. 如需生成代码，可按需修改根类型名称或对应语言选项
6. 右侧会自动展示转换结果
7. 你可以对左右两侧分别执行格式化、复制和下载

## 安全说明

1. 工具默认在当前浏览器页面内完成解析、转换和代码生成
2. 不会主动将输入内容上传到服务器
3. 上传文件仅在当前页面内存中读取
4. 若你主动执行复制或下载，后续数据将由浏览器和操作系统接管

## 限制说明

- `XML` 与对象结构之间并不存在唯一映射，属性、文本节点和数组结构可能与原文表现不同
- `TypeScript`、`Zod`、`Java`、`Go`、`C` 当前更适合作为结构草稿，而不是直接替代人工建模
- 右侧编辑不会反向同步到左侧
- 左侧切换输入类型会清空当前内容

## 常见问题

### 为什么我切换左侧输入类型后内容被清空了

因为左侧是主输入区，切换输入类型代表你在切换“当前数据语义”。直接清空可以避免把旧格式文本误认为新格式输入。

### 为什么 XML 转出来的结构和原文不完全一样

`XML` 属性、文本节点、空标签和重复节点在转成对象结构时通常需要做归一化处理，因此更适合保证“结构可读”和“语义接近”，而不是保证原文逐字符一致。

### 右侧生成的代码能直接用于生产环境吗

更建议把右侧结果当作结构草稿或起始模板。尤其是 `Java`、`Go`、`C`，通常还需要你根据项目规范补充注解、tag、指针或内存管理逻辑。
